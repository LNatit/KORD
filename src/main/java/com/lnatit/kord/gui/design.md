# Minecraft 按键绑定冲突智能判定系统 - 主界面 UI 设计规格书（最终版）

## 1. 全局约束
- 界面最小支持分辨率为 **320×240** 像素。
- 窗口实际尺寸若小于 320×240，界面不应加载，可绘制提示文字“分辨率不足”。
- 窗口尺寸 ≥ 320×240 时，界面自动适配，区域尺寸严格按规则动态计算。
- **所有按钮都不实现具体功能**，仅提供明确的回调方法声明（如 `void onXxx()`），供后续绑定。
- 列表内容、颜色、文本等均使用占位数据填充，确保 UI 预览成立，数据生成逻辑不得写入 UI 类中。

## 2. 整体布局及宽度计算
界面宽度分为左、中、右三个主区域，高度铺满窗口：

| 区域 | 宽度计算公式 (窗口宽度 W >= 320) |
|------|--------------------------------|
| **左侧操作边栏** | `min( W * 10%, 32px )` |
| **中间按键绑定列表** | `(W - 左侧宽度) * 5/9` |
| **右侧冲突信息面板** | `(W - 左侧宽度) * 4/9` |

- 当 W = 320：左侧 32px，中间 160px，右侧 128px。
- 当 W > 320：左侧固定 32px 不再变大，额外宽度按 5:4 分配给中间和右侧。
- 三区高度均为窗口高度 H。

**区域间距与内边距：**
- 左侧边栏：无内边距（padding = 0）。
- 中间区域：四周内边距 **4px**。
- 右侧面板：四周内边距 **4px**，且内部上下两部分（筛选+列表 与 详情+操作）之间保留 **4px 垂直间距**。

## 3. 通用视觉与行为规范
- **字体**：使用 Minecraft 原版像素字体（`fontRenderer`），字号 8-10 px。
- **背景规则**：
    - 左侧边栏：**深灰色不透明**长条（如 `0xFF404040`），铺满全高。
    - 中间区域：**浅灰色半透明**背景（如 `0x80AAAAAA`），配合 4px padding。
    - 右侧面板：与中间相同的浅灰半透明背景，内部细节通过不同明度区分层级。
- **按钮样式**：
    - 默认**不绘制背景**，仅显示内容（图标或文字）。
    - 鼠标悬浮时**叠加一层深灰色半透明矩形**（如 `0x60404040`）。
    - 无边框、无高光，完全扁平化。
- **列表视图**：
    - 每行高度固定 **20px**，条目间**无空隙**。
    - 选中行背景变为蓝色半透明高亮（如 `0x803366FF`）。
    - 超出可视区域时支持鼠标滚轮滚动。

## 4. 左侧操作边栏
- 背景：深灰不透明，无 padding。
- **顶部按钮**：退出按钮，尺寸 24×24，图标为透明底“✕”，悬浮有半透明层。回调：
    - `void onExitClick()`
- **底部按钮组**：从边栏最底部向上依次排列 24×24 功能按钮，间距 2px。每个按钮仅显示透明底图标。悬浮有半透明层。回调：
    - `void onToolButtonClick(int index)`，`index` 从 0（最底部按钮）开始递增。
- **Tooltip**：所有按钮支持鼠标长悬显示 Tooltip（暂用占位文本；实现时可预留 `void onRequestTooltip(String text)` 接口）。

## 5. 中间按键绑定列表
- 背景：浅灰半透明，四周留白 4px。
- 标题：可选的 12px 高“按键绑定”标题栏（若无空间可省略）。
- **列表主体**：垂直滚动列表，行高 20px，无间隙。每行水平只分为**左右两栏**，不再有三栏。

| 左栏 | 右栏 |
|------|------|
| **按键名称** | **绑定按钮（含冲突指示）** |
| 文本，缩写键名（如“前进”），左对齐，不响应点击 | 显示当前绑定键名（如“W”），并**在按钮上叠加显示冲突严重度指示** |

- **绑定按钮设计细节**：
    - 宽度尽量大到可容纳键名与冲突指示，约占该行 50% 宽度。
    - 键名文字居中或左对齐显示在按钮内，用像素字体绘制。
    - **冲突严重度指示**整合于按钮上：可在按钮右上角绘制一个 **3×3 或 4×4 像素的彩色小方块**，颜色含义：
        - 无色/绿：无冲突
        - 黄：轻微冲突
        - 橙：警告
        - 红：严重冲突
    - 小方块应不被文字完全遮盖，优先右上角对齐。
    - 或者按钮整体背景色根据冲突等级变化（无冲突不绘制背景，冲突时叠加对应的半透明颜色层），但必须同时保留键名绘制。**两种方式任选其一，但推荐使用角标方式，阅读性更好。**
- **按钮交互**：
    - 鼠标左键点击按钮：回调 `void onBindingLeftClick(int keyIndex)`
    - 鼠标右键点击按钮：回调 `void onBindingRightClick(int keyIndex)`
    - 右键响应区限制在按钮右侧 50% 区域，防误触。
- **列表选中**：点击名称区域或按钮区域整行，选中该按键绑定，行高亮，并触发 `void onKeySelected(int keyIndex)`。

## 6. 右侧冲突信息面板
整体内边距 4px，内部垂直分为四个子区域：筛选条、冲突列表、冲突详情、底部操作区。

### 6.1 筛选条（顶部）
- 高度 20px，水平并排三个 Toggle 按钮，宽度等分。
- 文字：“严重”、“警告”、“轻微”，可缩写。
- 状态：按下（视觉凹陷，叠加深色层）表示显示该等级冲突；弹起（正常）表示隐藏该等级。初始全部按下。
- 点击切换，回调：`void onFilterToggle(int level, boolean active)`
    - level: 0=严重，1=警告，2=轻微
    - active: 切换后的状态

### 6.2 冲突条目列表
- 位于筛选条下方，占据该面板剩余高度约 60%。
- 垂直滚动列表，行高 20px，无间隙。
- 每行显示冲突的两个键名，如 `W ⇄ ↑`，并用文字颜色粗显等级（与角标颜色对应）。
- 选中某行时高亮，触发 `void onConflictSelected(int conflictIndex)`。

### 6.3 冲突详情显示区
- 位于冲突列表下方，占用约 25% 高度（与列表间无额外间隙，仅靠区域 4px 内边距分隔）。
- 显示当前选中冲突的多行描述文字（占位示例：“在菜单界面中，这两个按键可能同时触发。”）。
- 文本可自动换行，字体略小（9px），区域背景可稍暗以区分。

### 6.4 底部操作区
- 位于详情区下方，高度 24-28px。与上方保留 **4px 垂直间距**。
- 水平放置两个等宽按钮：**索引按钮**（左）、**排除按钮**（右），按钮间距 2px。
- **索引按钮**：
    - 显示文字“定位”或双箭头图标。
    - 严格区分鼠标左右键：
        - 左键：索引到冲突的第一个按键 → `void onLocateLeftKey(int conflictIndex)`
        - 右键：索引到冲突的第二个按键 → `void onLocateRightKey(int conflictIndex)`
    - 如框架难以区分，可降级为左键索引第一键、Shift+左键索引第二键，但描述中必须注明备选方案。
- **排除按钮**：
    - 显示文字“排除”。
    - 左键点击时**弹出二级菜单**（浮层），菜单包含占位项：
        - “忽略此冲突”
        - “自定义排除规则…”
        - “从社区导入…”
    - 点击其他区域或选择后菜单关闭。菜单项回调：`void onExcludeMenuAction(int actionId)`，actionId 按菜单项顺序从 0 开始。
- 所有按钮默认无背景，悬浮时半透明深灰叠层。

## 7. 绘制顺序
1. 原版 GuiScreen 默认背景。
2. 左侧深灰不透明边栏（全高）。
3. 中间浅灰半透明区域背景（含 4px padding）。
4. 右侧浅灰半透明背景。
5. 右侧内部分割：筛选条、冲突列表、详情区、操作区背景（可微调灰度）。
6. 各列表行文本、绑定按钮及冲突角标、按钮悬浮层、选中高亮。
7. 二级菜单（最顶层）。

## 8. 技术要求
- **必须继承** `net.minecraft.client.gui.GuiScreen`。
- 使用原版绘制方法：`drawRect`、`drawCenteredString`、`drawString` 等。
- 列表可用原版 `GuiScrollingList` 或自行管理滚动偏移量。
- 所有回调方法定义为 `protected` 空方法，并写清注释，供后续 override 实现。
- 提供占位数据方法 `getMockKeyBindings()` 和 `getMockConflicts()`，返回模拟列表，保证界面可运行。

# 模态检查层 (Modality Layer) 判断原理

在 Suck 键位冲突检测管线中，**模态检查层（第 7 层）**严格遵循“正交化架构”原则。本层**绝不干涉任何业务逻辑**（如“能否一边开火一边打开背包”，这是 ActionRoot 层的职责），而是纯粹聚焦于：**当不同模态的操作绑定在同一按键时，对玩家操作可达性（Reachability）及状态复原成本（Recovery Cost）的影响。**

## 1. 基础模态定义
我们从时间和状态机维度，将所有按键输入抽象为四种基础模态（`P, H, T, C`）：
* **无状态物理行为 (Stateless)**：
    * `P` (PRESS) - 单点瞬发：按下瞬间触发，无持续时间。
    * `H` (HOLD) - 长按持续：按下时维持，松开时结束，占用一段物理时间。
* **持久逻辑状态 (Stateful)**：
    * `T` (TOGGLE) - 开关切换：二元状态机（开/关）。
    * `C` (CYCLE) - 循环轮转：多元状态机（大于2个状态的循环，如模式切换）。

## 2. 冲突分类体系与复原成本
基于组合碰撞后产生的“额外操作数（即玩家纠正状态所需的额外按键次数）”，我们将冲突严格划分为 5 大类别（OM, TM, RS, SL, SE），并依此推导严重度（Severity）。

### 🟢 OM - Operation Match (操作契合)
* **组合**：`PP`, `HH`
* **定级**：**`SAFE`**
* **原理**：物理与逻辑时序完全同频，不产生任何错位，是完美的键位复用。

### 🔵 TM - Timing Mismatch (时序不符)
* **组合**：`PH`, `HT`, `HC`
* **定级**：`PH`, `HT` = **`INFO`** | `HC` = **`WARNING`**
* **原理**：`HOLD` 具有独特的物理时间轴占用特性。当它与其他操作绑定时，必然导致时序交叠。
    * **HT (Hold+Toggle)**：连带触发。若想复原 Toggle 状态，玩家仅需**额外敲击 1 次**，成本极低，定级为 `INFO`。
    * **HC (Hold+Cycle)**：连带循环。若想复原 Cycle 的 $N$ 个状态，玩家需额外敲击 **$N-1$ 次（复数操作）**，干扰显著增加，定级提升为 `WARNING`。

### 🔵/🟡 RS - Repeat Switch (连带切换)
* **组合**：`PT`, `PC`
* **定级**：`PT` = **`INFO`** | `PC` = **`WARNING`**
* **原理**：高频的无状态敲击（PRESS）意外驱动了状态机。
    * **PT (Press+Toggle)**：二元状态翻转，复原成本为 1，定级 `INFO`。
    * **PC (Press+Cycle)**：多元状态轮转，高频点击极易导致状态迷失，复原成本为复数，定级 `WARNING`。

### 🟡 SL - State Lock (相位锁死)
* **组合**：`TT`
* **定级**：**`WARNING`**
* **原理**：两个二元状态机绑定在同一按键。一旦因外部因素（如死亡、GUI内单独点击）导致状态错位，它们将进入“永久反相（Permanent Anti-phase）”。玩家**永远无法通过该共用按键使二者重新对齐**，操作陷入局部死锁。

### 🔴 SE - State Explode (状态爆炸)
* **组合**：`TC`, `CC`
* **定级**：**`SEVERE`**
* **原理**：不同周期的状态机绑定（例如 2态与 3态、3态与 4态）。一旦发生错位，若要使其重新对齐，玩家必须敲击这两个周期的**最小公倍数次（LCM）**。在实际游玩中，这等同于该按键的状态机完全崩溃（Unreachable），产生灾难性的操作体验。

---

“虽然在闭环图论中，M-SL 导致的错位具有无限大的单键复原成本（不可达），但由于其二元反相的特性具有极高的认知可预测性，且存在特殊玩法的可能性，故将其定级控制在 WARNING，以区分于认知完全崩溃的 SEVERE 级状态爆炸。”
